## 注意事项

### 开发约束

1. **不使用 Redis** - 当前阶段不使用缓存，所有查询直接访问数据库
2. **数据爬取** - 需要编写爬虫脚本，注意反爬策略
3. **按顺序开发** - 严格遵循 22 个阶段，不跳步骤
4. **遵循规范** - 所有后端开发必须遵循 `backend-api-standards.md`

### 数据获取注意事项

- 目标网站：下厨房、美食杰
- 需要处理 IP 代理和 User-Agent 轮换
- 需要下载并存储图片
- 需要清洗和标准化数据

### 数据可视化重点

- 普通用户：只读统计图表（饼图、柱状图、雷达图）
- 管理员：深度分析（多维度、趋势、转化率）
- 用户行为：点击流、活跃用户、页面停留分析
- 数据大盘：关键指标 + 可交互图表

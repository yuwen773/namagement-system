# 菜谱数据分析系统 产品需求文档 (PRD)

## 1. 项目概况
本项目旨在构建一个集菜谱展示、收藏与深度数据分析于一体的综合平台。系统不仅为普通用户提供丰富的菜谱探索体验，还为管理员提供强大的数据管理与可视化分析能力，通过处理万级规模的菜谱数据，实现对饮食趋势、食材关联及用户行为的深度洞察。

### 1.1 项目目标
- **数据规模**：支撑 10,000 - 20,000 条真实/模拟菜谱数据，包含高清图片。
- **可视化核心**：提供高性能、交互性强的数据大盘，具备深度分析能力。
- **用户体验**：简洁直观的 UI/UX，确保在处理大数据量时依然流畅。

---

## 2. 角色定义与权限
| 角色 | 描述 | 核心权限 |
| :--- | :--- | :--- |
| **普通用户** | 菜谱的使用者与探索者 | 浏览、查询、分类查看、个人中心、查看公共数据报表 |
| **管理员** | 系统的维护者与数据分析师 | 系统设置、用户管理、菜谱数据增删改查、深度行为分析、大盘配置 |

---

## 3. 功能需求

### 3.1 角色一：普通用户模块

#### 3.1.1 账户与个人中心
1.  **注册与登录**：支持手机号/邮箱注册，提供标准的登录校验流程。
2.  **个人信息管理**：支持修改昵称、个人简介等。
3.  **安全管理**：提供密码修改功能，确保账户安全。

#### 3.1.2 菜谱探索与互动
1.  **菜谱查询**：支持关键字、食材、标签等多维度模糊搜索。
2.  **分类浏览**：按菜系（川、粤、鲁等）、场景（早餐、晚餐等）、人群等标签分类展示。
3.  **菜谱详情**：展示成品图、所需食材、详细步骤、难度系数、烹饪时长等信息。
4.  **热门推荐**：基于点击量或收藏量展示 Top 菜谱。
5.  **快捷收藏**：在列表页或详情页提供一键收藏功能。

#### 3.1.3 收藏夹功能
1.  **我的收藏列表**：展示用户已收藏的菜谱，支持按时间或分类排序。
2.  **取消收藏**：支持从收藏列表中移除不再需要的菜谱。

#### 3.1.4 数据展示（只读）
1.  **菜系分布图**：通过饼图展示全平台菜系的占比情况。
2.  **难度/口味统计**：通过柱状图或雷达图展示菜谱的难度分布和口味偏好。
3.  **食材频率统计**：通过词云或 Top 榜单展示高频使用的食材。

### 3.2 角色二：管理员模块

#### 3.2.1 系统与权限管理
1.  **用户账户管理**：查看用户信息，支持封禁/解封违规账户。
2.  **角色权限分配**：配置不同层级管理员的操作权限（可选）。

#### 3.2.2 基础数据管理
1.  **菜谱管理**：支持对菜谱数据的批量导入（CSV/JSON）、手动新增、编辑与删除。
2.  **分类/标签管理**：灵活定义菜谱分类、口味标签、难度等级等。
3.  **食材库管理**：维护食材基础信息，支持食材关联分析。

#### 3.2.3 数据分析中心（核心）
1.  **多维度分析**：
    - **菜系分析**：区域流行趋势分析。
    - **难度/口味分布**：识别平台内容的偏向性。
    - **热门菜谱分析**：深度挖掘点击与收藏背后的规律。
    - **食材关联分析**：分析食材之间的搭配频率。
2.  **数据趋势分析**：按日/周/月展示菜谱更新趋势、用户增长趋势。

#### 3.2.4 数据可视化仪表盘
1.  **总览大盘**：实时展示平台总数据量、今日新增、活跃用户数等关键指标。
2.  **用户行为分析**：
    - 模拟/统计用户登录频次、页面停留时间。
    - **点击流分析**：分析用户从搜索到收藏的转化路径。
3.  **分析图表**：集成 ECharts/AntV 等工具，提供可交互、下钻的图表。

---

## 4. 数据要求

### 4.1 数据量级
- **核心数据**：最少 10,000 条，建议 20,000 条。
- **数据来源**：
    - 爬取开源美食网站（如：下厨房、美食杰）。
    - 使用公开的菜谱数据集。
- **图片要求**：每条菜谱必须包含至少一张成品缩略图，详情页建议包含步骤图。

### 4.2 模拟数据（用户行为）
- 管理员端需展示“用户行为统计”，由于系统初期可能缺乏真实用户，需通过脚本模拟：
    - 模拟 100+ 用户在不同时间段的登录与浏览。
    - 模拟收藏、搜索等交互行为，以填充分析大盘。

---

## 5. 非功能性需求

### 5.1 性能要求
- **加载速度**：万级数据下的搜索响应时间应小于 500ms。
- **可视化性能**：图表渲染不卡顿，支持大数据量的聚合展示。

### 5.2 UI/UX 设计
- **风格**：清新、现代，以美食图片为视觉中心。
- **交互**：响应式布局，适配 PC 端（首选）及移动端。

### 5.3 安全性
- 管理员后台需严格的权限校验，防止数据泄露。

---

## 6. 技术架构建议 (参考)
- **前端**：Vue 3 / React + Tailwind CSS + ECharts/AntV
- **后端**：Python (Django/FastAPI) 或 Node.js (NestJS)
- **数据库**：MySQL (主存储)
- **数据处理**：Pandas/NumPy (用于后台数据分析统计)
- ⚠️ **注意**：当前阶段不使用 Redis 缓存

# 菜谱数据分析系统 技术栈说明文档 (Tech Stack)

本项目采用前后端分离架构，结合强大的数据处理能力，旨在构建一个高性能、可扩展的菜谱分析平台。

## 1. 后端技术栈 (Backend)

### 1.1 核心框架
- **Django 5.2**: 高级 Python Web 框架，提供快速开发和干净、务实的设计。利用其强大的 ORM 和管理后台。
- **Django REST Framework (DRF)**: 用于构建 Web API 的强大且灵活的工具包，实现前后端标准 JSON 数据交互。

### 1.2 数据库
- **MySQL**: 关系型数据库管理系统，用于持久化存储菜谱、用户、收藏及系统配置等核心业务数据。

### 1.3 身份验证与安全
- **JWT (JSON Web Token)**: 用于前后端分离架构下的用户身份认证。
- **Django Middleware**: 用于权限控制和请求日志记录。

---

## 2. 前端技术栈 (Frontend)

### 2.1 核心框架
- **Vue 3**: 渐进式 JavaScript 框架，采用 Composition API (组合式 API) 提升代码复用性与可维护性。

### 2.2 UI 组件库
- **Element Plus**: 基于 Vue 3 的桌面端组件库，用于快速构建整洁、高效的管理员后台和用户交互界面。
- **Tailwind CSS**: 原子化 CSS 框架，用于实现快速的自定义布局和响应式设计，减少 CSS 代码冗余。

### 2.3 数据可视化
- **ECharts**: 强大的可视化图表库，用于展示菜系分布、热门趋势、用户行为统计等核心分析数据。

### 2.4 状态管理与路由
- **Pinia**: Vue 的官方状态管理库，简洁且类型安全。
- **Vue Router**: 官方路由管理器，支持单页面应用 (SPA) 的导航逻辑。

---

## 3. 数据处理与分析 (Data Processing)

### 3.1 核心库
- **Pandas**: 提供高性能、易用的数据结构（如 DataFrame）和数据分析工具。用于后台对万级菜谱数据进行清洗、转换和聚合统计。
- **NumPy**: 基础科学计算库，支持大规模多维数组与矩阵运算，作为 Pandas 的底层支撑，提升计算性能。

### 3.2 应用场景
- **食材频率统计**: 利用 Pandas 进行词频统计。
- **口味与难度分析**: 对非结构化或半结构化数据进行分类汇总。
- **用户行为建模**: 对模拟的用户点击、搜索数据进行趋势预测和关联规则分析。

---

## 4. 开发与部署建议

### 4.1 包管理
- **Backend**: 使用 `pip` 或 `poetry` 管理依赖，并生成 `requirements.txt`。
- **Frontend**: 使用 `npm` 或 `pnpm` 作为包管理工具。

### 4.2 环境隔离
- 使用 `venv` 或 `Conda` 创建 Python 虚拟环境。
- 使用 Docker (可选) 进行容器化部署，确保开发与生产环境一致。

# 网络问答平台信息采集系统 - 产品需求文档 (PRD)

## 1. 文档介绍
### 1.1 文档目的
本文档旨在详细定义“基于Python的网络问答平台信息采集系统”的功能需求、用户界面规划及系统行为，作为开发、测试和验收的依据。

### 1.2 文档范围
本系统主要针对 [360问答](https://wenda.so.com/) 平台进行数据采集，提供数据清洗、存储、管理及可视化分析功能。系统分为管理端和用户端。

---

## 2. 产品概述
### 2.1 背景与目标
随着网络问答数据量的激增，人工整理效率低下且难以保证数据的实时性和准确性。本系统旨在构建一套自动化的数据采集与管理平台，通过 Python 爬虫技术自动获取问答数据，并提供直观的可视化分析和便捷的管理界面，以提高信息利用效率。

### 2.2 用户角色
| 角色 | 描述 | 权限 |
| :--- | :--- | :--- |
| **管理员** | 系统维护者 | 拥有所有权限，包括启动爬虫、管理所有数据、管理用户账号。 |
| **普通用户** | 数据使用者 | 拥有数据浏览、检索、查看分析图表的权限，不可操作爬虫和管理其他用户。 |

---

## 3. 功能需求

### 3.1 认证与个人中心模块
*   **登录**: 所有用户需登录后方可进入系统。
*   **个人中心**:
    *   查看基本信息（用户名、角色）。
    *   修改登录密码。
*   **注销**: 安全退出系统。

### 3.2 数据采集模块 (核心)
*   **爬虫控制 (演示友好型)**:
    *   **“开始爬取”按钮**: 仅管理员可用。
    *   **采集模式**: 
        *   默认执行**演示采集**（如：仅采集最新 20 条数据），确保演示时点击按钮能快速完成并看到效果。
        *   后台需具备**全量采集**能力，用于补充历史数据。
    *   **状态反馈**: 显示当前爬虫状态（空闲、运行中、已完成、异常）。
*   **采集策略**:
    *   目标网站: `https://wenda.so.com/`
    *   数据内容: 问题标题、问题描述、回答内容、回答时间、回答者、标签等。
    *   **数据量验收标准**: 
        *   系统交付时，数据库中需预置不少于 **10,000 条** 有效数据。
        *   系统架构（数据库索引、查询接口）需通过 1万条以上数据的性能测试，确保不卡顿。
*   **数据清洗**: 入库前自动去除 HTML 标签、去重、处理异常字符。

### 3.3 数据管理模块
*   **数据列表**:
    *   分页展示采集到的问答数据。
    *   显示关键字段：标题、发布时间、回答数等。
*   **数据检索**: 支持按标题关键词模糊搜索。
*   **数据详情**: 点击列表项，通过弹窗或抽屉方式展示完整的问题与回答详情。
*   **数据删除**: 仅管理员可删除无效或错误的数据记录。

### 3.4 数据可视化模块
*   **统计图表**:
    *   基于数据库中的海量数据（10,000+条）生成分析图表。
    *   **示例指标**:
        *   热门话题/标签词云。
        *   每日/每周问答数量趋势图（折线图）。
        *   高频回答者排名（柱状图）。
*   **展示位置**: 集成在系统首页/仪表盘。

### 3.5 用户管理模块 (仅管理员)
*   **用户列表**: 查看系统内所有注册用户。
*   **用户编辑**: 重置用户密码、修改用户角色。
*   **用户删除**: 删除非法用户。
*   **用户新增**: 管理员可直接创建新用户。

## 4. 页面规划 (精简设计)

为了满足“最少前端页面”且“不增加负担”的要求，系统采用**单页应用 (SPA)** 结构。

### 4.1 页面数量统计
*   **管理端**: 共 **5** 个页面 (符合 "五六个就行" 的要求)。
*   **用户端**: 共 **4** 个页面 (符合 "四五个" 的要求)。

### 4.2 页面详细清单

| 页面名称 | 对应路由 | 适用角色 | 核心功能 |
| :--- | :--- | :--- | :--- |
| **1. 登录页** | `/login` | 全部 | 用户名/密码登录。 |
| **2. 仪表盘 (首页)** | `/dashboard` | 全部 | **管理员**: 数据统计图表 + **爬虫控制卡片** (点击“开始爬取”演示采集功能)。<br>**用户**: 仅展示数据统计图表。 |
| **3. 数据中心** | `/data` | 全部 | 问答数据表格、搜索栏、详情查看 (弹窗)、删除 (仅管理员)。 |
| **4. 个人中心** | `/profile` | 全部 | 修改密码、查看个人信息。 |
| **5. 用户管理** | `/users` | **仅管理员** | 用户列表、添加用户、编辑/删除用户。 |

*(注：数据详情通过弹窗展示，不计为独立页面)*

### 4.3 导航结构 (侧边栏)
*   **Dashboard / 仪表盘**
*   **Data Center / 数据中心**
*   **User Management / 用户管理** (普通用户不可见)
*   **Profile / 个人中心**

## 5. 非功能需求

### 5.1 性能要求
*   **大数据量支持**: 数据库设计需针对 10,000+ 条数据进行优化（如添加索引），确保列表查询和搜索响应时间不超过 2秒。
*   **爬虫稳定性**: 爬虫脚本需具备错误重试机制，防止因网络波动导致中断。

### 5.2 易用性
*   **操作反馈**: 点击“开始爬取”后，界面应立即给出“正在采集...”的提示，采集完成后自动刷新数据或提示“采集完成”。

### 5.3 安全性
*   **密码存储**: 密码采用明文存储 (无需加密)。
*   API 接口需进行身份验证 (Token 或 Session)。

# 汽车改装件销售推荐平台产品需求文档 (PRD)

## 1. 文档介绍
### 1.1 文档目的
本文档旨在明确“基于Django的汽车改装件销售推荐平台”的功能需求、非功能需求及系统设计约束，为开发团队、测试团队及项目管理人员提供统一的参考标准。

### 1.2 项目背景
本项目旨在构建一个专注于汽车改装件销售的电商平台，集成个性化推荐功能，连接管理员与普通用户。平台需支持丰富的改装件商品展示、流畅的交易流程以及精准的营销推广，同时满足教学演示或小规模运营的数据模拟需求。

## 2. 总体说明
### 2.1 用户角色
| 角色 | 描述 | 预期规模 |
| :--- | :--- | :--- |
| **管理员 (Platform Admin)** | 负责平台运营、商品管理、订单处理及系统维护。 | 2-3 个账号 |
| **普通用户 (User)** | 浏览商品、下单购买、评价及售后。 | 约 20 个账号（需模拟大量交互数据） |

### 2.2 核心业务流程
1.  **商品上架**：管理员发布改装件信息（含图片、属性）。
2.  **用户浏览与推荐**：用户浏览商品，系统根据规则或算法推荐相关商品。
3.  **交易闭环**：用户下单 -> 支付（模拟） -> 管理员发货 -> 用户收货 -> 评价。
4.  **售后服务**：用户申请退换货 -> 管理员审核处理。

## 3. 功能需求
### 3.1 管理员端 (Platform Admin)
*注意：模块顺序需严格遵循以下定义。*

#### 3.1.1 商品管理模块
- **商品信息维护**：支持改装件商品的新增、编辑、删除。
    - *必填项*：商品名称、分类、价格、库存、详情描述、商品图片（支持多图）。
- **商品分类管理**：维护商品的多级分类结构。
- **商品属性管理**：定义不同类型商品的规格属性（如车型适配、材质、颜色）。
- **库存管理**：查看及调整商品库存数量。
- **商品审核与上下架**：控制商品的前台可见状态（上架/下架/待审核）。

#### 3.1.2 用户与权限管理模块
- **用户列表管理**：查看注册用户信息（昵称、手机号、注册时间）。
- **用户详情查看**：查看特定用户的收货地址簿、当前积分状态。
- **账号状态控制**：对违规或异常账号进行禁用/解禁操作，保障平台安全。

#### 3.1.3 订单与交易管理模块
- **订单管理**：
    - 列表展示：查看所有用户订单，支持按状态（待付款、待发货、已发货、已完成、已取消）筛选。
    - 详情查看：查看订单包含的商品、收货信息、支付金额。
    - 发货处理：录入物流信息（物流公司、单号）进行发货。
- **退换货处理**：
    - 查看用户的退换货申请。
    - 审核操作：同意或拒绝申请，并记录处理意见。
- **交易记录查询**：提供平台维度的交易流水查询与统计功能。

#### 3.1.4 营销活动管理模块
- **优惠券管理**：
    - 创建优惠券（满减、折扣）。
    - 设置有效期、发放总量、每人限领数量。
- **促销活动设置**：配置特定时间段的商品促销规则。
- **营销素材管理**：管理Banner图、活动海报等素材。
- **营销统计**：简单的活动效果分析（如优惠券核销率、活动期间销售额）。

#### 3.1.5 推荐管理模块
- **推荐规则配置**：设置推荐逻辑（如“热门推荐”、“新品推荐”或基于用户行为的协同过滤参数）。
- **推荐内容管理**：手动干预推荐列表，置顶特定推广商品。
- **推荐中心**：监控推荐系统的运行效果，确保商品曝光率。

#### 3.1.6 系统管理模块
- **改装案例管理**：发布和维护精选的汽车改装案例文章/图片，建立社区氛围。
- **知识库管理**：维护改装知识、常见问题解答（FAQ）。
- **系统参数配置**：平台基础设置（如网站名称、SEO设置）。
- **消息通知发布**：向用户发送站内信或系统公告。
- **操作日志记录**：记录管理员的关键操作行为（如删除商品、修改订单状态），便于审计。

### 3.2 普通用户端 (User)
#### 3.2.1 用户注册与登录
- **注册**：填写手机号/邮箱、密码、验证码完成账号创建。
- **登录**：支持账号密码登录。

#### 3.2.2 个人信息与账号安全管理
- **基本信息修改**：修改昵称、头像。
- **安全设置**：修改登录密码，绑定/换绑联系方式。

#### 3.2.3 收货地址管理
- **CRUD操作**：新增、修改、删除收货地址。
- **默认地址**：设置默认收货地址。

#### 3.2.4 商品浏览与评价
- **商品列表**：按分类、价格、销量筛选和排序商品。
- **商品详情**：查看商品大图、详细参数、库存及其他用户评价。
- **商品评价**：交易完成后，对购买的商品进行评分（1-5星）和文字评价，支持上传图片。

#### 3.2.5 订单管理
- **下单流程**：加入购物车 -> 确认订单（选择地址、优惠券） -> 提交订单。
- **订单列表**：查看不同状态的订单。
- **订单详情**：查看物流进度、订单金额明细。
- **取消订单**：在未发货前可取消订单。

#### 3.2.6 退换货申请
- **发起申请**：针对已收到且存在问题的商品，填写退换货原因、上传凭证图片，提交申请。
- **进度查询**：查看管理员的处理进度和结果。

#### 3.2.7 优惠券使用
- **领券中心**：领取平台发放的优惠券。
- **使用优惠券**：结算时选择可用优惠券抵扣金额。

## 4. 数据与性能需求
### 4.1 数据规模要求
- **改装件商品数据**：
    - **数量**：不少于 **1000条** 真实或高仿真的改装件数据。
    - **质量**：每条商品需包含完整属性及至少1张清晰图片。
- **用户模拟数据**：
    - **用户量**：预置约 20 个活跃用户账号。
    - **行为数据**：每个预置用户需包含多条浏览记录、购物车记录、订单记录及评价数据，以体现系统的活跃度和推荐算法的有效性。

### 4.2 非功能性需求
- **响应速度**：页面加载时间在正常网络环境下不超过 2秒。
- **安全性**：
    - 用户密码（当前开发阶段使用明文存储以简化调试，生产环境需升级为加密存储）。
    - 敏感操作需进行权限校验。
- **易用性**：界面简洁大方，符合电商用户操作习惯。

## 5. 技术栈建议
- **后端**：Python, Django, Django REST Framework (可选)
- **数据库**：MySQL / SQLite (开发阶段)
- **前端**：HTML/CSS/JS (Bootstrap/Tailwind) 或 Vue.js/React (前后端分离架构)
- **图片存储**：本地文件系统 或 对象存储 (OSS/S3)
